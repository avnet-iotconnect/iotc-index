# /IOTCONNECT® Enablement Hub

A static, marketing-grade catalog of the Avnet [/IOTCONNECT](https://www.avnet.com/iotconnect)
ecosystem — **pre-enabled development boards, SDKs, and ready-to-run demos** — browsable
by partner, board, topic and language, with per-board QuickStart guides, demos, videos
and buy links.

- **Live (staging):** https://mlamp99.github.io/iotc-index/
- **Destination:** to be moved into the [`avnet-iotconnect`](https://github.com/avnet-iotconnect) org
- **Hosting:** GitHub Pages, served straight from `main`

The entire site is generated from **one spreadsheet**. You edit the spreadsheet,
commit it, and a GitHub Action rebuilds the data the page renders.

---

## What the site does

- **Hero + partner showcase** — Avnet-branded landing with live counts and the five
  silicon partners (Infineon · Microchip · NXP · Renesas · STMicroelectronics).
- **Hardware view** — every enabled dev board as a card (image, vendor, part number,
  AWS/Greengrass badges, project count). Click any board to open a **detail drawer**
  with where-to-buy, QuickStart & Developer guides, demos, webinars, videos and blogs.
- **SDKs & Demos view** — the open-source repos across the org (SDKs, libraries, example
  projects) with language, topic and status filters.
- **By-partner view** — everything grouped by silicon vendor.
- Full-text search, deep-links (`#board/<slug>`, `#software`), keyboard-accessible
  cards/drawer, and a robust image fallback (Azure host → bundled local image → monogram).

## How it works (data flow)

```
iotc-index-catalog.xlsx     ← single source of truth (you edit this)
          │
          │  build_site.py  +  live GitHub facts (stars, last-updated)
          ▼
   index.json  +  AUDIT.md  ← generated artifacts (do not hand-edit)
          │
          ▼
      index.html            ← Avnet-branded front end; fetches index.json in the browser
          │
          ▼
   GitHub Pages (live site)
```

`.github/workflows/build-index.yml` runs `build_site.py` on every commit of the
workbook (and weekly) and commits the refreshed `index.json` + `AUDIT.md`.
`index.html` is never touched by CI.

## The workbook (`iotc-index-catalog.xlsx`)

Four data sheets (plus an `Instructions` sheet of notes):

### `Listings` — the SDK/demo cards
One row per listing. Columns: `Name`, `Type` (`sample`/`sdk`/`library`), `Status`,
`Repo`, `Languages`, `Topics`, `Boards` (one or more board **Part Numbers**),
`Description`, `Link`, `Include`, `Image`.
- A **`sample`** referencing several boards expands into **one card per board**.
- A **`library`/`sdk`** referencing several boards stays a **single card**.
- Leave `Boards` blank for platform-agnostic SDKs — then give the row an `Image`
  so the card still shows a relevant picture. `Image` accepts a tech logo
  (`assets/tech/python.svg`), a bundled photo (`assets/listings/drone.png`), or a
  full URL. Listings **with** boards use the board image automatically and ignore `Image`.

### `Boards` — the hardware registry
One row per dev board. Columns: `Manufacturer`, `Board Name`, `Part Number`,
`Image File` (Azure-hosted filename), **`Image Local`** (bundled fallback under
`assets/boards/`), `Product Link`, `AWS Qualified`, `Greengrass`, `Tags`, `Include`.

### `Resources` — per-board guides, demos & links *(new)*
The engine behind the board detail drawer. One row per link. Columns:
- `Ref` — a board **Part Number** (or a manufacturer name)
- `RefType` — `board` or `manufacturer`
- `Kind` — `buy` · `quickstart` · `developer` · `demo` · `webinar` · `video` · `blog` · `doc` · `info`
- `Title`, `URL`

The build groups these under each board; the first `buy` row becomes the board's
primary **Buy on Avnet** button.

### `Config` — key/value settings
`ORG`, `IMAGE_BASE` (Azure board-image host), `IMAGE_LOCAL_BASE` (`assets/boards/`),
`BRAND_BASE` (`assets/brand/`), `DEFAULT_STATUS`, `PAGES_URL`.

## Board images

Each board can resolve an image from three layers, in order:
1. **`Image File`** at the Azure `IMAGE_BASE` host (the curated photo set), or a full URL.
2. **`Image Local`** committed under `assets/boards/` (offline-safe; future-proof for the
   move into the avnet repo).
3. A clean vendor **monogram** placeholder if neither loads.

This makes the grid render reliably even if a remote host is down or a file is missing.

## Building locally

```bash
pip install openpyxl
python build_site.py            # reads the workbook, writes index.json + AUDIT.md
python -m http.server 8000      # then open http://localhost:8000/
```

Optional env: `GITHUB_TOKEN` (live stars/last-updated + uncatalogued-repo audit),
`CATALOG_XLSX`, `OUT_DIR`, `ORG`.

To re-audit link liveness: `python migration/_linkcheck.py`.

---

## `AUDIT.md` — the gap report

Regenerated on every build. Surfaces: boards referenced but undefined, boards with no
image, orphan boards, org repos with no listing (needs `GITHUB_TOKEN`), and listings
missing description/topics. The header reports live counts.

## Common tasks

- **Add/edit a board or listing** → edit the workbook, commit (CI rebuilds), or run
  `python build_site.py` locally and commit the regenerated `index.json`/`AUDIT.md`.
- **Add board guides/demos/buy links** → add rows to the `Resources` sheet keyed by the
  board's Part Number.
- **Add a board image** → drop the file in `assets/boards/` and set `Image Local` (and/or
  upload to the Azure host and set `Image File`).
- **Hide something without deleting** → set `Include = no`.
- **Check what's incomplete** → read `AUDIT.md`.
