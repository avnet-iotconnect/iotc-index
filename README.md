# iotc-index — /IOTCONNECT Hardware & SDK Index

A static catalog website that indexes the [avnet-iotconnect](https://github.com/avnet-iotconnect)
hardware boards, SDKs, and example projects, browsable by manufacturer and topic.

- **Live site:** https://mlamp99.github.io/iotc-index/
- **Hosting:** GitHub Pages (served straight from this repo's `main` branch)

The whole site is generated from **one spreadsheet**. You edit the spreadsheet,
commit it, and a GitHub Action rebuilds the data the page renders. There are no
per-repo manifests to maintain.

---

## How it works (data flow)

```
iotc-index-catalog.xlsx   ← single source of truth (you edit this)
          │
          │  build_site.py  +  live GitHub API facts (stars, last-updated)
          ▼
   index.json  +  AUDIT.md   ← generated artifacts (do not hand-edit)
          │
          ▼
      index.html             ← static front end, fetches index.json in the browser
          │
          ▼
   GitHub Pages (live site)
```

1. **`iotc-index-catalog.xlsx`** is the single source of truth. All content lives here.
2. **`build_site.py`** reads the workbook (plus live GitHub facts when a token is
   present) and writes `index.json` and `AUDIT.md`.
3. **`index.html`** is the front end. In the browser it fetches `index.json` and
   renders the cards, manufacturer/topic filters, and counts.
4. **`.github/workflows/build-index.yml`** runs the build automatically (see
   [CI](#automation-ci)), commits the refreshed `index.json` + `AUDIT.md`, and
   Pages serves the result.

## The workbook (`iotc-index-catalog.xlsx`)

Four sheets:

### `Listings` — the cards shown on the site
One row per listing (an SDK, library, or example project).

| Column | Meaning |
|---|---|
| `Name` | Display title of the card. |
| `Type` | `sample`, `sdk`, `library`, … Drives card behavior (see below). |
| `Status` | e.g. `beta`, `stable`. Defaults to Config `DEFAULT_STATUS` if blank. |
| `Repo` | GitHub repo name within the org (used to fetch live stars/last-updated). |
| `Languages` | Comma/semicolon-separated. Falls back to the repo's primary language. |
| `Topics` | Comma-separated filter tags (`ai`, `vision`, `lora`, `audio`, …). |
| `Boards` | One or more board **Part Numbers** (comma-separated) this listing targets. |
| `Description` | Card text. Falls back to the repo's GitHub description if blank. |
| `Link` | Optional explicit URL (otherwise derived from `Repo`). |
| `Include` | Set to `no` to hide the row without deleting it. |

Rendering rules:
- A **`sample`** referencing several boards expands into **one card per board**.
- A **`library`/`sdk`** referencing several boards stays a **single card** listing them all.
- Leave `Boards` blank for platform-agnostic libraries.

### `Boards` — the hardware registry
One row per dev board.

| Column | Meaning |
|---|---|
| `Manufacturer` | Vendor (becomes a manufacturer facet). `Other` is excluded from facets. |
| `Board Name` | Full product name. |
| `Part Number` | The key listings reference in their `Boards` column. |
| `Image File` | **Just the filename** (e.g. `MIC-710AIL.png`); or a full `http(s)://` URL. See [Board images](#board-images). |
| `Product Link` | Vendor product page. |
| `AWS Qualified` | `yes` → adds an "AWS Qualified" qualification badge. |
| `Greengrass` | `yes` → adds a "Greengrass" qualification badge. |
| `Tags` | Comma-separated board tags. |
| `Include` | Set to `no` to exclude the board. |

### `Config` — key/value settings
| Key | Current value | Purpose |
|---|---|---|
| `ORG` | `avnet-iotconnect` | GitHub org queried for live facts + repo audit. |
| `IMAGE_BASE` | `https://saleshosted.z13.web.core.windows.net/images/boards/` | Prefix prepended to bare `Image File` names (Azure static host). |
| `DEFAULT_STATUS` | `beta` | Status applied when a listing leaves `Status` blank. |
| `PAGES_URL` | `https://mlamp99.github.io/iotc-index/` | The live site URL. |

### `Instructions` — human-readable usage notes (not read by the build).

## Board images

Board photos are **not stored in this repo**. The `Image File` column holds a
bare filename that the build prepends with Config `IMAGE_BASE` (an Azure static
web host). For example `MIC-710AIL.png` resolves to:

```
https://saleshosted.z13.web.core.windows.net/images/boards/MIC-710AIL.png
```

Convention is `<PartNumber>.png`. A few entries use vendor-prefixed names
(`Dell_3200.png`) or a full external `http(s)://` URL instead — both are
supported (a value starting with `http:`, `https:`, `data:`, or `//` is used
as-is). **The image must actually be uploaded to the Azure host** for it to
render; writing a filename does not upload anything.

## Building locally

Requires Python 3 and `openpyxl`:

```bash
pip install openpyxl
python3 build_site.py
```

This reads `iotc-index-catalog.xlsx` and (over)writes `index.json` + `AUDIT.md`.

Optional environment variables:

| Var | Effect |
|---|---|
| `GITHUB_TOKEN` | Enables live GitHub facts (stars, last-updated) and the "uncatalogued repos" audit. Without it, the build still works using workbook data only. |
| `CATALOG_XLSX` | Override the input workbook path (default `iotc-index-catalog.xlsx`). |
| `OUT_DIR` | Override the output directory (default `.`). |
| `ORG` | Override the GitHub org (default from Config). |

To preview the site, serve the folder and open `index.html` (it `fetch`es
`index.json`, so it needs to be served over HTTP, not opened as a `file://`):

```bash
python3 -m http.server 8000   # then visit http://localhost:8000/
```

## Automation (CI)

`.github/workflows/build-index.yml` regenerates `index.json` + `AUDIT.md` and
commits them back. It runs:

- **On every push** that changes `iotc-index-catalog.xlsx`, `build_site.py`, or
  the workflow file — so editing the workbook and committing updates the site.
- **Weekly** (Mondays 07:00 UTC) to refresh live GitHub facts and re-audit, so
  stars/last-updated and the gap report stay current even with no edits.

It writes **only** `index.json` and `AUDIT.md`; `index.html` is never touched by CI.

> Note: `build_site.py` must be committed to the repo for CI to run.

---

## `AUDIT.md` — the gap report

Regenerated on every build. It surfaces data-quality gaps so the catalog stays
honest:

- **Boards referenced but missing from the Boards sheet** — a listing points at a
  Part Number that has no board definition (renders with no image/vendor info).
- **Boards with no image** — defined boards missing an `Image File`.
- **Boards not used by any listing (orphans)** — defined but unreferenced.
- **Org repos with no listing** — public org repos not yet catalogued (needs `GITHUB_TOKEN`).
- **Listings missing description or topics** — incomplete cards.

The header line also reports live counts (listings / boards in use / manufacturers).

## Common tasks

**Add or edit a listing or board**
1. Edit `iotc-index-catalog.xlsx` (Listings and/or Boards sheet).
2. Run `python3 build_site.py` (or just commit — CI will rebuild).
3. Commit the workbook (and the regenerated `index.json`/`AUDIT.md` if you built locally).

**Add a board image**
1. Upload the image to the Azure `boards/` host.
2. Put its filename in the board's `Image File` column.
3. Rebuild.

**Hide something without deleting it** — set `Include = no` on that row.

**Check what's incomplete** — read `AUDIT.md`.
